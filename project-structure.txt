PET-PLATFORM-BACKEND PROJECT STRUCTURE
========================================

Root Directory
--------------
├── server.js                      # Main server entry point
├── package.json                   # Dependencies and scripts
├── .gitignore                     # Git ignore rules
├── database.sql                   # Database schema and sample data
├── env.example                    # Environment variables template
├── README.md                      # Main project documentation
├── SETUP.md                       # Setup instructions
├── API_DOCUMENTATION.md           # Complete API documentation
└── project-structure.txt          # This file

Config Directory
----------------
└── config/
    └── database.js                # Database connection configuration

Controllers Directory
---------------------
└── controllers/
    ├── authController.js          # Authentication (register, login)
    ├── userController.js          # User CRUD operations
    ├── locationController.js      # Location CRUD operations
    ├── petController.js           # Pet CRUD operations
    ├── adoptionController.js      # Adoption request management
    ├── sittingController.js       # Pet sitting request management
    ├── bookingController.js       # Booking management
    ├── paymentController.js       # Payment management
    └── reviewController.js        # Review and rating management

Routes Directory

└── routes/
    ├── auth.js                    # Authentication routes

    ├── pets.js                    # Pet routes

    ├── sittings.js                # Pet sitting routes
    ├── bookings.js                # Booking routes
    ├── payments.js                # Payment routes

Middleware Directory
--------------------
└── middleware/
    └── auth.js                    # JWT authentication middleware

DATABASE TABLES
===============
1. Location          - Geographic locations
2. User             - Pet owners, sitters, or both
3. Pet              - Pet information and status
4. AdoptionRequest  - Pet adoption requests
5. SittingRequest   - Pet sitting requests
6. Booking          - Booking records
7. Payment          - Payment transactions
8. Review           - User reviews and ratings for pets, sitters, or both

API ENDPOINTS SUMMARY
=====================
Base URL: http://localhost:3000/api

Auth:
- POST /auth/register
- POST /auth/login

Users:
- GET    /users
- GET    /users/:id
- PUT    /users/:id
- DELETE /users/:id

Locations:
- GET    /locations
- GET    /locations/:id
- POST   /locations
- PUT    /locations/:id
- DELETE /locations/:id

Pets:
- GET    /pets
- GET    /pets/:id
- GET    /pets/owner/:ownerId
- POST   /pets
- PUT    /pets/:id
- DELETE /pets/:id

Adoption Requests:
- GET    /adoptions
- GET    /adoptions/:id
- POST   /adoptions
- PUT    /adoptions/:id
- DELETE /adoptions/:id

Sitting Requests:
- GET    /sittings
- GET    /sittings/:id
- POST   /sittings

- DELETE /sittings/:id

Bookings:
- GET    /bookings
- GET    /bookings/:id
- POST   /bookings
- PUT    /bookings/:id
- DELETE /bookings/:id

Payments:
- GET    /payments
- GET    /payments/:id
- POST   /payments
- PUT    /payments/:id
- DELETE /payments/:id

Reviews:
- GET    /reviews
- GET    /reviews/:id
- GET    /reviews/pets/:id

- DELETE /reviews/:id

TECHNOLOGY STACK
================
- Runtime:       Node.js
- Framework:     Express.js
- Database:      MySQL
- ORM:           None (raw SQL with mysql2)
- Auth:          JWT (jsonwebtoken)
- Password Hash: bcryptjs
- CORS:          cors
- Environment:   dotenv
- Dev Tool:      nodemon
